## What is FakeGuard AI?

FakeGuard AI is a machine learning system that classifies news articles as **real or fake** using two complementary signals:

1. **TF-IDF text features** вҖ” learns which words and phrases are statistically associated with fake vs. real journalism
2. **Knowledge Graph entity features** вҖ” builds a co-occurrence graph of named entities (people, organisations, places, dates, events) across the entire dataset and uses each article's position in that graph as additional evidence

The graph-enhanced model achieves **95.91% accuracy** while also being interpretable вҖ” you can see exactly which entities it flagged and how strongly they lean fake or real. The text-only baseline reaches **99.48%** with near-perfect precision.

## Model Overview

Two classifier variants were trained on **44,898 labeled news articles** (23,481 fake + 21,417 real):

| **Graph-Enhanced PAC** вң… deployed | TF-IDF + 5 graph features | **95.91%** | 0.9622 |

## Quickstart

### Prerequisites

| Requirement | Version |
|---|---|
| Python | 3.10+ |
| Node.js | 18+ |
| pip | any recent |

### Git LFS (required for knowledge graph)

The knowledge graph file is tracked via Git LFS. Make sure Git LFS is installed before cloning/pulling.

**Install and enable Git LFS:**
```bash
# Linux (Debian/Ubuntu)
sudo apt-get install git-lfs

git lfs install
```

```bash
# Windows (Chocolatey)
choco install git-lfs -y

git lfs install
```

**Clone and fetch LFS objects:**
```bash
# USE ONLY ONE FROM THESE TWO
git clone https://github.com/Kanika-Sharma205/fake-news-detection-using-nlp.git # via HTTP
git clone git@github.com:Kanika-Sharma205/fake-news-detection-using-nlp.git # via SSH      

cd fake-news-detection-using-nlp
git lfs pull
```

**If you already cloned without LFS:**
```bash
git lfs install
git lfs pull
```

### Step 1 вҖ” Generate model files

Run all cells in [`training/notebooks/Fake_News_Detection.ipynb`](training/notebooks/Fake_News_Detection.ipynb).

This trains both models and saves to `models/`:
- `tfidf_vectorizer.pkl`
- `pac_graph_model.pkl`
- `knowledge_graph.pkl` *(~112 MB вҖ” takes ~5 min to build)*

It also auto-exports real run metrics to `backend/metrics/model_metrics.json`.

> **Kaggle dataset setup** (one-time):
> ```bash
> mkdir -p ~/.kaggle
> cp ~/Downloads/kaggle.json ~/.kaggle/
> chmod 600 ~/.kaggle/kaggle.json   # Linux / macOS only
> ```

### Step 2 вҖ” Install backend dependencies

```bash
python -m venv venv
source venv/bin/activate          # Windows: venv\Scripts\activate
pip install -r backend/requirements.txt
python -m spacy download en_core_web_sm
```

### Step 3 вҖ” Run everything

**Linux / macOS вҖ” single command:**
```bash
bash run-linux.sh
```

**Windows вҖ” single command:**
```
run-windows.bat
```

This opens both servers simultaneously. Or run them separately:

```bash
# Terminal 1 вҖ” backend  (http://localhost:8000)
bash backend/start.sh             # Windows: backend\start.bat

# Terminal 2 вҖ” frontend  (http://localhost:5173)
bash frontend/start.sh            # Windows: frontend\start.bat
```

> вҸі **First startup is slow** вҖ” loading the 112 MB knowledge graph takes ~30 seconds.

## API Reference

Base URL: `http://localhost:8000`  
Interactive docs: `http://localhost:8000/docs`

### `GET /api/health`

Returns server + model status.

```json
{
  "status": "ok",
  "models_loaded": true,
  "model_type": "TF-IDF + Knowledge Graph + Passive Aggressive Classifier"
}
```

---

### `GET /api/metrics`

Returns the full metrics JSON from the last training run вҖ” accuracy, F1, confusion matrices, knowledge graph stats, top entities, dataset info, and training config.

---

### `GET /api/sample?type=real|fake|random`

Returns a random article from the dataset. Every call returns a different article.

| param `type` | Source |
|---|---|
| `real` | Random row from `True.csv` |
| `fake` | Random row from `Fake.csv` |
| `random` | Random from either file |

**Response:**
```json
{
  "headline": "Federal Reserve holds rates steady amid cooling inflation",
  "content":  "WASHINGTON (Reuters) вҖ” The Federal Reserve...",
  "label":    "real"
}
```

---

### `POST /api/predict`

Classifies an article as real or fake.

**Request:**
```json
{
  "text": "Federal Reserve holds rates steady... WASHINGTON (Reuters)..."
}
```

**Response:**
```json
{
  "label":    "REAL NEWS",
  "is_fake":  false,
  "confidence": 0.9934,
  "entities_detected": ["federal reserve", "washington", "reuters", "u.s."],
  "graph_features": {
    "num_entities": 4,
    "num_edges":    6,
    "avg_degree":   3.0,
    "density":      0.5,
    "fake_ratio":   0.14
  },
  "message": "This article appears to be legitimate news."
}
```

> The frontend concatenates headline + body as `headline + ' ' + content` before sending вҖ” matching exactly how training data was prepared (`title + ' ' + text`).

## Changelog

### v0 вҖ” Initial Release *(May 2026)*

**FakeGuard AI v0 is the first complete, fully working version of the project.**

#### ML Pipeline
- Trained on **44,898 labeled news articles** (23,481 fake В· 21,417 real) from the Kaggle Fake and Real News dataset
- **Text-Only PAC** baseline: TF-IDF (50,000 vocabulary) + Passive Aggressive Classifier вҶ’ **99.48% accuracy**
- **Graph-Enhanced PAC** (deployed): TF-IDF + 5 knowledge graph structural features вҶ’ **95.91% accuracy**
- Knowledge graph of **127,994 unique named entities** connected by **4,299,650 co-occurrence edges**, with per-node fake/real frequency counters
- Entity extraction via **spaCy `en_core_web_sm`** вҖ” PERSON, ORG, GPE, DATE, EVENT labels
- 5 graph features per article: `num_entities`, `num_edges`, `avg_degree`, `subgraph_density`, `entity_fake_ratio`
- Feature fusion using **SciPy sparse `hstack`** (50,000 TF-IDF + 5 graph = 50,005 total)

#### Backend (FastAPI)
- `POST /api/predict` вҖ” full inference pipeline with confidence score and graph feature breakdown
- `POST /api/debug` вҖ” raw decision score, class metadata, and SHA-256 model file fingerprints
- `GET /api/sample?type=real|fake|random` вҖ” pulls live articles from `Fake.csv` / `True.csv`
- `GET /api/metrics` вҖ” exports full training metrics (accuracy, F1, confusion matrix, top entities, dataset info)
- `GET /api/health` вҖ” model load status

#### Frontend (React + Vite + Tailwind)
- **Hero** section with live stats (44K articles, 128K graph nodes, model accuracy)
- **Problem Statement** вҖ” why fake news detection matters
- **Model Approach** вҖ” 6-step pipeline walkthrough
- **Performance Dashboard** вҖ” accuracy bar chart, radar charts, confusion matrices, entity fake-ratio bars, and training visualisation PNGs
- **Live Predictor** вҖ” headline + body input, рҹҹў Real / рҹ”ҙ Fake / рҹҺІ Random buttons, result card with detected entities and graph feature breakdown

#### Deployment & Tooling
- Deployed on **Hugging Face Spaces** (Docker, free tier) вҖ” FastAPI serves the built frontend as static files
- GitHub Actions workflow: **auto-push to HuggingFace** on every push to `main`
- GitHub Actions cron job: **daily ping** to keep the HF Space from going to sleep
- `run-linux.sh` / `run-windows.bat` вҖ” one-command local startup (auto-detects venv, frees port 8000, starts both servers with colour-coded logs)

#### Bug Fixes (caught and resolved during v0)
- **Entity extraction mismatch** вҖ” backend was producing 2вҖ“3Г— more entities than training (no deduplication, no `len > 2` filter). Fixed to exactly match the notebook's `extract_entities_from_doc`: set-deduplication + `.strip()` + length filter. This was causing highly confident wrong predictions (e.g. FAKE articles predicted as REAL NEWS with 100% confidence).
- **`num_entities` off-by-definition** вҖ” was counting all spaCy-detected entities instead of only those present in the knowledge graph. Fixed to count only graph-matched entities, matching training-time `len(valid)`.
- **Terminal log colour escape codes** вҖ” `run-linux.sh` used single-quoted `'\033[...]'` strings which bash treats as literals. Fixed to `$'\033[...]'` so ANSI colours render correctly.
- **Port conflict on restart** вҖ” `run-linux.sh` would crash with "address already in use" if a previous backend was still running. Fixed by adding a `lsof -ti:8000 | xargs kill -9` cleanup step before starting.
